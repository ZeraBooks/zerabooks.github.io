AI Transaction Categorization

Zera Books AI categorization takes the transactions extracted from bank statements and maps them to chart of accounts categories for QuickBooks Online or Xero — automatically, without manual coding rules.

This is what separates Zera Books from basic converters. Most PDF-to-Excel tools stop at extraction. Zera Books continues through categorization, so what you get back is ready to import, not ready to manually code.

How Categorization Works

The AI reads the transaction description — for example, "AMZN Mktp US*1A2B3C4D" — and maps it to the appropriate chart of accounts category. Common vendor names, payroll descriptions, and recurring transaction patterns are recognized from training data. The system also applies pattern matching on description text to identify likely categories for transactions it hasn't explicitly seen.

Each categorization result includes a confidence score. High-confidence categorizations (above the threshold) are applied automatically. Low-confidence ones are flagged so you can review just those transactions rather than auditing everything.

Pattern Learning

When a categorization is corrected — for example, you change "Office Supplies" to "Software Subscriptions" for Adobe transactions — the system learns that pattern for future batches from the same account. Recurring transactions from known vendors get categorized consistently without repeated manual corrections.

What This Replaces

Without AI categorization, the workflow is: extract transactions to Excel, create VLOOKUP rules or manually review each row, assign account codes, re-import. For a statement with 300 transactions, that's a lot of manual work. Zera Books compresses it to reviewing flagged exceptions only.
