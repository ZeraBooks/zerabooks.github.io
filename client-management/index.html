Batch Processing Workflow

Month-end processing typically involves the same set of clients, each with 1-5 statement PDFs. The batch processing workflow is designed for this pattern — upload everything for the month in one job, get organized output for each client.

1

Upload all files at once

Drag and drop PDFs from all clients into the batch upload interface. Files can be named anything — the platform identifies document type and content automatically.

2

Assign to client folders

Files are automatically associated with client folders based on your naming configuration, or manually assigned if preferred. Multi-account PDFs are split and assigned to the same client.

3

Parallel processing

All documents process simultaneously — a 50-file batch doesn't take 50x the time of a single file. Most batches complete in under 5 minutes regardless of size.

4

Organized output download

Download output by client, or export all at once. Each output file is named with the source document name and processing date for easy filing.

Month-end time savings: A 30-client firm that individually uploads 90 statements (3 per client) and manually assigns each to a client folder spends 45+ minutes on file management. Frequently Asked Questions

How many clients can I manage in Zera Books?

Unlimited. The $79/month plan covers unlimited clients with no per-client fees. No billing penalty for practice growth.

How does batch processing work?

Upload up to 50+ documents simultaneously via drag-and-drop. Files are processed in parallel. Output organized by client and document type. Most batches complete in under 5 minutes.

Is conversion history kept indefinitely?

Yes. Conversion history is retained indefinitely and output files can be re-downloaded at any time. Uploaded source PDFs are retained for 30 days per security policy, then auto-deleted.
