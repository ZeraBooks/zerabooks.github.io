Document Types and Extraction Specs

The decision to support four document types instead of one is deliberate. Accounting workflows don't stop at bank statements — firms also reconcile invoices, prepare financial reports, and process check runs. Switching between four different tools for four document types adds friction and error. Here's what each type extracts:

Document TypeFields ExtractedInput AcceptedOutput Formats
Bank StatementsDate, description, debit, credit, balance, account number, opening/closing balanceDigital PDF, scanned PDF, JPG, PNGExcel, CSV, QBO, IIF
Financial StatementsLine items, period headers, account codes, subtotals, totals, comparative columnsDigital PDF, scanned PDFExcel, CSV
InvoicesVendor, date, invoice number, line items, quantities, unit prices, tax amounts, PO numberDigital PDF, scanned PDF, JPG, PNGExcel, CSV
ChecksMICR routing number, account number, check number, payee, amount, date, memo fieldScanned PDF, JPG, PNGExcel, CSV

Multi-account detection: A single PDF containing checking + savings + credit card data is automatically split into separate output files. See multi-account detection for methodology details.
