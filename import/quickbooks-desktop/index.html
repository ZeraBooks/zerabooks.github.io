IIF vs QBO for QuickBooks Desktop Imports

QuickBooks Desktop supports two formats for bank statement imports. Choosing the right one saves significant troubleshooting time.

QBO (Web Connect)Banking > Web ConnectAuto-maps by account numberRecommended — same format banks use
IIFFile > Utilities > ImportMaps to existing QB accountsBulk historical imports, custom structures

Use QBO format for most imports. QuickBooks Desktop's Web Connect reads QBO files the same way it reads bank feeds — the account number in the QBO header auto-matches to your register. IIF is better for historical imports where you need to specify category/account for each transaction. Zera Books exports both QBO and IIF formats for QuickBooks Desktop.

IIF File Structure for QuickBooks Desktop

IIF (Intuit Interchange Format) is a tab-delimited format with a header row and data rows. The structure is strict — wrong field names or missing required columns cause the entire import to fail.

FieldRequiredFormatNotes
!TRNS / TRNSRequiredHeader / Data row marker!TRNS defines columns, TRNS contains data
DATERequiredMM/DD/YYYYQuickBooks Desktop uses US date format
ACCNTRequiredAccount name in QBMust exactly match an existing account name
AMOUNTRequiredSigned decimalPositive for deposits, negative for payments
MEMORecommendedTextTransaction description/payee
!ENDTRNS / ENDTRNSRequiredEnd markerEvery transaction group must end with ENDTRNS

ACCNT must match exactly. The account name in the IIF ACCNT field must exactly match an existing account in your QuickBooks company file — including capitalization and spaces. Zera Books IIF export includes an account configuration step where you specify your QuickBooks account names to ensure perfect matching.

2

Export QBO or IIF for QuickBooks Desktop

In the export panel, choose "QuickBooks Desktop (QBO)" or "QuickBooks Desktop (IIF)" depending on your preferred import method. For most users, QBO is recommended — it's faster to import and doesn't require specifying account names.

If choosing IIF, configure your QuickBooks account name in Zera Books settings first. The ACCNT field in the IIF output must match your QuickBooks register account name exactly.
3

Import QBO via Banking → Web Connect

In QuickBooks Desktop: go to Banking in the top menu, then Import Web Connect File. Navigate to your QBO file and open it. QuickBooks asks which account to link — select the matching bank register account.

QuickBooks Desktop Web Connect reads the account number from the QBO header. If your account number matches an existing register, it auto-selects. If not, you'll manually choose the destination account from the dropdown.
4

OR: Import IIF via File → Utilities → Import

For IIF: go to File > Utilities > Import > IIF Files. Select your IIF file. QuickBooks imports all TRNS records and confirms how many transactions were imported.

QuickBooks Desktop IIF import gives a success count but no detailed error log for failed rows. If the count is lower than expected, check the IIF file for rows where ACCNT doesn't match an existing account.
5

Review imported transactions in the register

After import, open the bank account register (Banking > Use Register or Chart of Accounts > double-click account). Review imported transactions, categorize any uncategorized items, and run a bank reconciliation.

Transactions imported via IIF may not initially appear in the register if the import confirmation count was zero. Verify the register account name in QuickBooks matches the ACCNT field in your IIF exactly.

Common Errors and Fixes

These are the most frequent issues and their specific solutions.

'Invalid format' error on IIF import

The IIF file has a malformed header, missing !ENDTRNS markers, or tab character issues. Fix: Re-export from Zera Books using the QuickBooks Desktop IIF option — it generates validated IIF with correct headers and ENDTRNS markers for every transaction group.

Transactions import to wrong account

The ACCNT field in the IIF doesn't match any QuickBooks account, so QB defaults to a catch-all account. Fix: Check the exact account name in QuickBooks (Chart of Accounts) and update your Zera Books account mapping to match.

QBO import shows 'Already Up-to-Date' — no transactions imported

The transaction dates in your QBO file fall within a date range already covered by the existing bank feed or previous import. QuickBooks Desktop won't import duplicates. Check the last transaction date in your register and create a QBO file covering only the gap period.

IIF transactions appear but amounts are reversed

Deposits are showing as negative and payments as positive. The AMOUNT sign convention is inverted. For QuickBooks Desktop IIF: deposits should be positive, payments should be negative. Re-export from Zera Books or check the bank statement debit/credit interpretation.

Frequently Asked Questions

What formats does QuickBooks Desktop accept for bank statement imports?

QuickBooks Desktop accepts QBO files via Web Connect (Banking > Import Web Connect File) and IIF files via File > Utilities > Import. Zera Books exports both formats from any PDF bank statement.

Is QBO or IIF better for QuickBooks Desktop?

QBO is generally faster and easier — it imports via Web Connect without requiring you to specify account names. IIF gives more control over transaction categorization but requires the ACCNT field to exactly match your QuickBooks account names.

Can I import a PDF bank statement into QuickBooks Desktop?

No. Convert the PDF to QBO or IIF format using Zera Books first. QBO is recommended for most users — import via Banking > Import Web Connect File in QuickBooks Desktop.

Does Zera Books support older QuickBooks Desktop versions?

Yes. Zera Books generates QBO and IIF files compatible with QuickBooks Desktop Pro, Premier, and Enterprise versions from 2016 onward. Both formats use long-standing specifications that haven't changed significantly across versions.
