Oracle Financials Bank Statement Import: Step-by-Step Guide

Oracle Fusion Financials and Oracle EBS handle bank statement imports through the Cash Management module. The standard format is BAI2, though Oracle also supports SWIFT MT940 and configurable CSV loaders. BAI2 is the format most US banks use for electronic statement delivery.

Oracle Fusion Cash Management Import Steps

  1. Go to Cash Management work area
  2. Select Bank Statements > Import Bank Statements
  3. Upload your BAI2 or MT940 file
  4. Run the Load Interface File for Import process if using SFTP load
  5. Run Import Bank Statements from the Scheduled Processes
  6. Review in Manage Bank Statements for any exceptions

BAI2 Record Structure

Record CodeDescription
01File header (sender, receiver, date, time)
02Group header (bank ID, currency)
03Account header (account number, currency, opening balance)
16Transaction (type code, amount, date, text)
49Account trailer (control total)
98Group trailer
99File trailer

Each bank uses different transaction type codes in record 16. Common codes: 195 (debit), 295 (credit), 399 (opening balance), 901 (total debits), 910 (total credits). Oracle maps these during the import validation step.

Common Import Errors

  • Control total mismatch: The account trailer (49) must sum all transactions. If even one digit is off, Oracle rejects the entire file.
  • Unknown transaction type code: If Oracle doesn't recognize a bank's type code, transactions get exception status. Map them in the Bank Transaction Codes setup.
  • Date format: BAI2 uses YYMMDD format. Some banks use YYYYMMDD. Oracle usually handles both but check your bank's specific format.

Frequently Asked Questions

What format does Oracle use for bank statement imports?

Oracle Fusion Cash Management and Oracle EBS primarily use BAI2 format for bank statement imports. Oracle also supports SWIFT MT940 and custom CSV via configurable bank statement loaders.

What is the BAI2 file format for Oracle?

BAI2 is a structured text format developed by Bank Administration Institute. It uses record type codes (01=file header, 02=group header, 03=account header, 16=transaction, 49=account trailer, 98=group trailer, 99=file trailer). Each bank uses slightly different type codes for transactions.

How do I import a PDF bank statement into Oracle Financials?

Oracle cannot read PDFs. Convert the PDF to BAI2 or CSV format first, then use Oracle's Cash Management > Bank Statements > Import to load the file. Zera Books outputs Excel and CSV exports compatible with Oracle's configurable import templates.
