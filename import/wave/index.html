Wave CSV Import Format Requirements

Wave Accounting accepts only CSV files for manual transaction imports. Unlike QuickBooks Online or Xero, Wave does not support QBO, OFX, or IIF format. This means any PDF bank statement must be converted to Wave's specific CSV structure before import.

Common Wave Import Errors and Fixes

Wave's CSV import error messages are often vague. These are the most common issues and their specific fixes.

"Something went wrong" — no specific error message

Wave rejects files with missing header rows, wrong column names, or BOM encoding. Fix: Re-export from Zera Books using the Wave CSV option. This ensures correct UTF-8 encoding and exact column names. If the error persists, open the CSV in a text editor and confirm the first row is exactly: Date,Description,Amount

Transactions import but show wrong amounts (all positive)

Wave expects Credits as positive and Debits as negative in a single Amount column, or uses separate Credit/Debit columns. If you're using a raw bank CSV, check how your bank exports amounts. Zera Books normalizes to the correct signed-amount format Wave expects.

Date format rejected

Wave accepts YYYY-MM-DD and MM/DD/YYYY but rejects DD/MM/YYYY (European format). If you're manually editing a CSV, convert all dates to MM/DD/YYYY. Zera Books standardizes date output automatically for the Wave export option.

Duplicate transactions after import

You imported a date range that overlaps with transactions already in Wave from bank sync or previous imports. Check Wave's last synced transaction date and only import transactions from the gap period not covered by existing data.

Frequently Asked Questions

What file format does Wave accept for bank statement imports?

Wave only accepts CSV files for manual imports. It does not support QBO, OFX, or PDF formats. Zera Books exports Wave-formatted CSV from any PDF bank statement.

How do I import a PDF bank statement into Wave?

Wave cannot read PDFs. Convert the PDF to CSV first using Zera Books or similar tool, then import via Accounting > Transactions > Import transactions. The CSV must have Date, Description, and Amount columns with a header row.

Why does Wave reject my CSV import?

Most Wave CSV rejections are caused by: missing header row, wrong column names, incorrect encoding, or European date format (DD/MM/YYYY). Re-export from Zera Books using Wave CSV format to fix all of these at once.

Can I import multiple bank accounts into Wave at once?

Wave's import process handles one account at a time. However, Zera Books can batch-convert 50+ statements and auto-separate multi-account PDFs into individual CSV files — one per account — so you just import each file sequentially into the matching Wave account.
