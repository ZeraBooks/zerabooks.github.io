Common Xero Import Errors and Fixes

'Date format not recognized' error

Xero expects DD/MM/YYYY for UK/AU/NZ orgs and MM/DD/YYYY for US orgs. Zera Books detects your region and formats dates accordingly — this error doesn't occur with Zera Books exports.

'Unable to map columns' error

Generic CSV exports use headers like "Transaction Date" or "Debit/Credit" instead of Xero's expected "Date" and "Amount." The Zera Books Xero preset uses the exact header names Xero requires.

Duplicate transactions after import

Overlapping date ranges between bank feeds and CSV imports create duplicates. Zera Books' duplicate detection removes overlapping transactions before export.

Amounts showing as wrong sign

Xero treats positive amounts as money in and negative as money out. Some converters use separate debit/credit columns instead of signed values. Zera Books consolidates to a single signed Amount column matching Xero's convention.

Frequently Asked Questions

Can I import a PDF bank statement directly into Xero?

No. Xero accepts CSV, OFX, and QIF files. Convert your PDF to Xero-formatted CSV using Zera Books, then import via Accounting > Bank Accounts > Import a Statement.

What CSV format does Xero require?

Date, Amount, Payee, Description, and Reference columns. Date must match your Xero region (DD/MM/YYYY for UK/AU/NZ, MM/DD/YYYY for US). Amount must be signed. Zera Books generates this automatically.

Does Zera Books handle scanned PDF statements for Xero?

Yes. Zera OCR processes scanned and image-based PDFs with 95%+ accuracy, then converts to Xero-ready CSV. Scan quality of 300+ DPI produces the best results.

Can Zera Books separate multi-account PDFs for Xero import?

Yes. When a single PDF contains multiple accounts, Zera Books generates separate CSV files for each. Import each into the matching Xero bank account.
