Excel to Xero: Bank Statement Import Guide

Xero does not accept Excel files. The import path is always: prepare your Excel data, save as CSV, then upload to Xero. The most common failures come from date format mismatches and Excel-specific formatting that doesn't translate cleanly to CSV.

Preparing the Excel File

  1. Delete non-transaction rows: Remove account summaries, opening/closing balances, page headers, and blank rows
  2. Format dates correctly: Select the date column > Format Cells > Custom. Enter the format matching your Xero region (DD/MM/YYYY for UK/AU/NZ, MM/DD/YYYY for US)
  3. Clean the amount column: Remove currency symbols and thousands separators. Use Format Cells > Number with 2 decimal places
  4. Check debit/credit direction: Xero expects negative amounts for money out. If your bank shows debits as positive, create a helper column that multiplies debits by -1
  5. Keep only needed columns: Date, Description/Payee, Amount (or Spent/Received)

Saving as CSV from Excel

File > Save As > change file type to CSV UTF-8 (Comma delimited). Use UTF-8 specifically — the plain "CSV (Comma delimited)" option can produce encoding issues with special characters in bank descriptions (ampersands, accented characters, etc.) that Xero will reject.

Xero Import Steps

  1. In Xero: Accounting > Bank Accounts > click your account
  2. Click Import a Statement
  3. Upload the CSV file
  4. On the column mapping screen, set the date format to match your file
  5. Map: Date, Amount (or Spent/Received), Payee, Description
  6. Check the transaction count in the preview — should match your file
  7. Click Import

Frequently Asked Questions

Can Xero import Excel files directly?

No. Xero does not accept XLSX or XLS files. You must save as CSV (comma-separated) first, then upload via Accounting > Bank Accounts > Import a Statement.

How do I prepare an Excel bank export for Xero?

Remove header/footer rows, keep only Date, Amount (or Spent/Received), Payee, Description columns. Format dates as DD/MM/YYYY or MM/DD/YYYY to match your Xero region. Save as CSV. Remove any currency symbols from amounts.

Why does Xero show import errors after uploading my Excel CSV?

Common causes: date format mismatch (check your Xero organisation date setting), amounts with currency symbols, blank rows parsed as transactions, or the file encoding. Try saving as UTF-8 CSV from Excel.
