CSV to QuickBooks: Formatting Guide and Import Steps

Getting a CSV file into QuickBooks requires the right column structure. QuickBooks is particular about format — a misplaced column header or wrong date format will cause the import to silently fail or misread your data.

QuickBooks Online CSV Column Format

The minimum valid CSV for QuickBooks Online looks like this:

DateDescriptionAmount
01/15/2025AMAZON MKTP US-52.99
01/16/2025PAYROLL DEPOSIT3200.00
01/17/2025OFFICE SUPPLIES-128.40

Key rules:

  • Use negative amounts for withdrawals/debits, positive for deposits/credits
  • No currency symbols — -52.99 not -$52.99
  • No commas in amounts — 3200.00 not 3,200.00
  • Date format must match your QuickBooks regional setting
  • No blank rows between transactions

Alternative: Debit/Credit Column Format

If your bank exports separate debit and credit columns, you can use that format too:

DateDescriptionDebitCredit
01/15/2025AMAZON MKTP US52.99
01/16/2025PAYROLL DEPOSIT3200.00

Leave the unused column blank — don't put a zero, as QuickBooks may treat it as a $0 transaction.

Import Steps in QuickBooks Online

  1. Go to Banking > Upload transactions (or use the link icon next to your account)
  2. Select Upload a file
  3. Choose your CSV file
  4. Map columns: QuickBooks will show a preview — verify Date, Description, Amount are mapped correctly
  5. Set the date format if prompted
  6. Click Next and review the transaction preview
  7. Click Yes to import, then review transactions in the For Review tab

Common Formatting Mistakes

  • File saved as XLSX: QuickBooks reads CSV only. In Excel, use File > Save As > CSV (Comma delimited).
  • BOM character: Some Excel exports add a byte-order mark at the start of the file that confuses QuickBooks. Open in Notepad and delete any characters before "Date" in the first line.
  • Trailing commas: Rows like 01/15/2025,AMAZON,-52.99, with a trailing comma can cause column mapping errors.

Frequently Asked Questions

What columns does QuickBooks need in a CSV import?

QuickBooks Online CSV imports require at minimum: Date, Description, and Amount. Optionally you can include separate Debit and Credit columns instead of a signed Amount. Column headers must be on the first row.

What date format does QuickBooks CSV import accept?

QuickBooks Online accepts MM/DD/YYYY, DD/MM/YYYY (UK accounts), or YYYY-MM-DD. The format must match your QuickBooks account's regional date format setting, or dates will be misread.

Why does my QuickBooks CSV import keep failing?

The most common causes: wrong date format, currency symbols in the Amount column (use plain numbers only), blank rows in the middle of data, or the file saved as XLSX instead of CSV. Open in Notepad to check the raw format.
