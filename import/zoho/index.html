Zoho Books Bank Statement Import: Step-by-Step Guide

Zoho Books supports importing bank statements as OFX, CSV, or TSV files. It also has a bank feeds feature connecting to many banks directly, but the manual import is needed for banks without a feed, or to load historical data.

Import Steps in Zoho Books

  1. Go to Banking in the left sidebar
  2. Click the bank account you want to import into
  3. Select Import Statement from the top menu
  4. Choose your file type (OFX, CSV, or TSV)
  5. Upload the file and map columns if using CSV
  6. Preview transactions, then click Import

If you don't include a Transaction Type column, use a signed Amount instead (negative for withdrawals, positive for deposits). Don't include both — Zoho will get confused about direction.

OFX Import

OFX is the cleanest option for Zoho Books. Upload via the same Import Statement screen and Zoho reads the file structure directly with no column mapping. The FITID field in OFX acts as a unique transaction ID, preventing reimport of the same transaction.

After Import

Imported transactions land in the Uncategorized section of your bank account in Zoho Books. From there, you can match them against open invoices, create rules for recurring payees, or manually code them. Zoho's auto-categorization learns from your patterns over time.

Frequently Asked Questions

What file formats does Zoho Books accept for bank imports?

Zoho Books accepts OFX, CSV, and TSV files for bank statement imports. OFX is preferred as it preserves transaction IDs for duplicate prevention.

How do I import a PDF bank statement into Zoho Books?

Zoho Books cannot process PDFs. Convert the PDF to OFX or CSV first, then import via Banking > your bank account > Import Statement. Zera Books handles the PDF-to-OFX conversion automatically.

What is the Zoho Books CSV import column order?

Zoho Books CSV requires: Date (MM/DD/YYYY), Payee Name (optional), Transaction ID (optional), Transaction Type (debit/credit), Amount. Column headers must match exactly.
