Sage Import Requirements by Version

Sage has several products with different import paths and format requirements. Know which version you're using before attempting an import.

Sage VersionImport PathFormatDate Format
Sage 50Banking > Import StatementCSV / OFXDD/MM/YYYY (UK) or MM/DD/YYYY (US)
Sage 200Banking > Bank Transactions > ImportCSVDD/MM/YYYY
Sage Business CloudBanking > Import StatementCSV / OFXDD/MM/YYYY
Sage IntacctCash Management > ImportCSVMM/DD/YYYY

Check your regional settings first. Sage uses your regional settings to determine date format. A UK-installed Sage expects DD/MM/YYYY; a US-installed Sage expects MM/DD/YYYY. Importing with the wrong date format causes dates to be misread or the file to be rejected. Zera Books Sage export lets you specify your Sage region before downloading.

Required CSV Columns for Sage Import

Sage CSV imports require at minimum three columns. The minimum structure that works across all Sage versions:

ColumnRequiredNotes
DateRequiredDD/MM/YYYY for UK, MM/DD/YYYY for US. No time stamps.
DescriptionRequiredTransaction narrative/memo. Sage uses this for auto-matching rules.
AmountRequiredSingle signed column: negative for payments out, positive for payments in.
ReferenceOptionalBank reference number or check number.
BalanceOptionalRunning balance. Used for reconciliation verification.

Common Sage Import Errors and Fixes

Date format mismatch

Most common Sage import error. UK Sage expects DD/MM/YYYY; US Sage expects MM/DD/YYYY. Importing with the wrong format causes inverted month/day. Fix: Use the Zera Books Sage export and select your region (UK or US) before downloading.

Column count error

Sage rejects CSV files where the number of columns varies row-to-row. This happens when description fields contain commas (e.g., "Amazon, Inc."). Fix: Zera Books wraps all text fields in quotes during export to prevent comma-in-description from breaking column count.

Amounts importing as wrong sign

All transactions importing as debits or all as credits. Sage expects a single signed Amount column — negative for money out, positive for money in. If your PDF had separate Debit/Credit columns, they need to be merged with correct sign convention before import.
