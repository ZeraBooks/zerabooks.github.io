Sage Bank Statement Import: Step-by-Step Guide

Sage 50, Sage 200, and Sage Intacct each have different import paths. Sage 50 uses a structured CSV format with transaction type codes. Sage Intacct has a native bank statement module with its own requirements.

Sage 50 Import Steps

  1. Open Sage 50 and go to Bank accounts
  2. Select the account, then click Statement > Import
  3. Or use File > Import > Transactions for a full CSV import
  4. Select your CSV file and verify the column mapping preview
  5. Check the audit trail after import to confirm posting

Sage Intacct Import

Sage Intacct's bank reconciliation module accepts CSV files. Navigate to Cash Management > Bank Accounts > select account > Import Statement. Required fields: Bank Transaction ID, Date, Amount, Description. The Bank Transaction ID must be unique across all imports for that account.

Common Errors

  • Wrong date format: Sage 50 requires DD/MM/YYYY. A US-formatted file (MM/DD/YYYY) will post transactions to wrong dates or throw an error for invalid dates above 12.
  • Missing nominal codes: If the nominal code doesn't exist in your chart of accounts, Sage will reject the line. Use a suspense code (9998 or similar) if you're not sure and recode later.
  • Locked periods: Transactions dated before a locked year-end cannot be imported. Change the date or unlock the period first.

Frequently Asked Questions

What file format does Sage 50 use for bank imports?

Sage 50 uses a CSV format with specific columns: Type, Account Reference, Nominal A/C Ref, Department Code, Date, Reference, Details, Net Amount, Tax Code, Tax Amount. The date must be in DD/MM/YYYY format.

Can I import PDF bank statements into Sage?

Sage cannot read PDFs directly. You need to convert the PDF to Sage-compatible CSV format first. Zera Books exports pre-formatted CSV files ready for Sage 50 import without manual column mapping.

What is the Sage 50 bank import transaction type code?

For bank imports in Sage 50, use transaction type BP (Bank Payment) for money out and BR (Bank Receipt) for money in. Using the wrong type code causes the transaction to post to the wrong side of the account.
