MYOB Bank Statement Import: Step-by-Step Guide

MYOB AccountRight and MYOB Essentials both support bank statement imports, but the process differs slightly. MYOB primarily uses OFX format — the same format used by Australian and New Zealand banks for their statement downloads.

MYOB AccountRight Import Steps

  1. Go to Banking menu > Bank Transactions
  2. Click Import Statement
  3. Select the bank account from the dropdown
  4. Choose your OFX or CSV file
  5. MYOB previews the transactions — review dates and amounts
  6. Click Import to add to the bank transaction list
  7. Allocate transactions in the Bank Transactions window

MYOB Essentials Import

In MYOB Essentials (online), go to Banking > select your account > Import statement. The process is similar but the interface is web-based. MYOB Essentials accepts both OFX and CSV. CSV column requirements:

ColumnFormat
DateDD/MM/YYYY (Australian format)
DescriptionText — appears as transaction memo
AmountPositive for credits, negative for debits

Australian Bank CSV Downloads

Most Australian banks export CSV files with their own column structure that doesn't match MYOB directly. Common issues:

  • ANZ: Exports with separate Debit/Credit columns — combine into a single signed Amount
  • CommBank: Includes a Balance column MYOB ignores — remove it to avoid import confusion
  • Westpac: Date format is sometimes DD-MM-YYYY (dashes, not slashes) — MYOB may reject this
  • NAB: Exports include a transaction type code column — remove before importing into MYOB

Frequently Asked Questions

What file format does MYOB use for bank statement imports?

MYOB AccountRight accepts OFX files for bank statement import. MYOB Essentials (the online version) also accepts CSV. OFX is the standard format for MYOB bank feeds and manual statement imports.

How do I import a PDF bank statement into MYOB?

MYOB cannot read PDF files. You must convert the PDF to OFX or CSV first, then import via Banking > Bank Transactions > Import Statement. Zera Books converts Australian and New Zealand bank statements to MYOB-compatible OFX.

Why is my MYOB bank import showing transactions on the wrong side?

MYOB uses a signed amount convention where credits (money in) are positive and debits (money out) are negative. If your CSV has debits as positive, transactions will post to the wrong side. Reverse the sign on your amount column before importing.
