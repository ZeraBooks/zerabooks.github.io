Why CSV Format Matters for Imports

CSV is the most universally accepted import format. QuickBooks, Xero, Sage, Wave, and virtually every accounting tool accept CSV imports. For firms with custom workflows — uploading to a database, running through a reconciliation script, feeding into a reporting tool — CSV is the format that works everywhere without format conversion.

Common Data Quality Problems (Avoided)

When banks export their own CSV files, the formatting is inconsistent. One bank uses MM/DD/YYYY, another uses DD-Mon-YYYY. One puts debits as negative numbers in a single "amount" column; another uses separate debit/credit columns with positive values. Parentheses for negatives in some formats. Zera Books normalizes everything to a consistent schema regardless of source bank.

Encoding and Delimiter

  • Encoding: UTF-8 with BOM (for Excel compatibility)
  • Delimiter: comma
  • Line endings: CRLF (Windows-compatible)
  • Quote character: double-quote, only when needed
  • Header row: always included
