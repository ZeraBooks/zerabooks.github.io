Data Extraction Pipeline

The Zera Books data extraction pipeline is the sequence of automated steps that transforms a raw PDF into clean, structured, categorized transaction data. Each step runs without manual intervention.

Six automated steps from upload to output. OCR → structural parsing → field extraction → data cleaning → deduplication → categorization.

Pipeline Steps

  1. Input detection — The system checks whether the uploaded file has a text layer (digital PDF) or is image-only (scanned PDF). Mixed files are flagged at the page level.
  2. OCR (scanned pages only) — Zera OCR processes image-only pages, converting them to text. MICR font recognition applies to check documents.
  3. Structural parsing — Zera AI identifies the document type, locates transaction tables, identifies header rows, column positions, and section boundaries. No templates used.
  4. Field-level extraction — Individual fields are extracted: date, description, debit, credit, balance, account identifier, and document-type-specific fields.
  5. Data cleaning and normalization — Dates normalized to YYYY-MM-DD. Amounts converted to numeric. Descriptions cleaned. Currency detected and labeled. Multi-line descriptions joined.
  6. Duplicate detection — Transactions matching on date, amount, and description across overlapping statement periods are flagged and optionally removed.
  7. AI categorization — Each transaction description is mapped to a chart of accounts category with a confidence score.
  8. Output generation — XLSX, CSV, QBO, IIF, or pre-formatted exports produced and stored in the client folder.

Data Cleaning Details

IssueHow It's Handled
Inconsistent date formatsNormalized to YYYY-MM-DD regardless of source
Amounts with currency symbolsStripped to numeric value; currency labeled separately
Comma-formatted amounts (1,234.56)Converted to plain numeric (1234.56)
Parentheses for negatives (1,234.56)Converted to standard negative format
Multi-line transaction descriptionsJoined into single field with delimiter
Excess whitespace in descriptionsCollapsed to single space
Bank abbreviations for known vendorsExpanded where recognized (e.g., "AMZN" → "Amazon")

Duplicate Detection Logic

Duplicates occur when a client provides overlapping statement periods — January's statement plus a Q1 summary that includes January again. Exact-match deduplication fails when description formatting differs slightly between documents. Zera Books uses fuzzy matching on the description field combined with exact matching on date and amount, so minor formatting differences don't create false negatives.

Frequently Asked Questions

What steps does Zera Books apply during data extraction?

Zera Books runs: (1) OCR for scanned pages, (2) AI-based structural parsing, (3) field-level extraction, (4) data cleaning and normalization, (5) duplicate detection, and (6) AI categorization. All steps are automated.

How does Zera Books handle multi-currency bank statements?

Zera Books detects currency symbols and codes in transaction amounts. Each transaction is tagged with its currency in the output. Multi-currency statements are extracted with currency labels preserved (not auto-converted).

Does Zera Books clean up transaction descriptions?

Yes. Zera Books standardizes transaction descriptions by removing excess whitespace, normalizing capitalization, and resolving common bank truncation patterns for known vendors.
