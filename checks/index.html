Check Processing and MICR Extraction — Zera Books

Zera Books check processing extracts structured data from scanned physical checks and check images. The MICR line at the bottom of each check — routing number, account number, check number — is read using OCR specifically trained on the E-13B MICR font.

Check processing is rare in this category. Most bank statement conversion tools don't handle physical checks at all. Zera Books processes checks alongside bank statements, invoices, and financial statements in the same platform.

Input Formats

  • Scanned check images (JPG, PNG)
  • Scanned check PDFs (one or multiple checks per page)

  • Batch processing: 50+ checks per upload

Output

Check data is output to Excel or CSV with one row per check. The output includes all MICR fields plus the handwritten/printed data from the check face. This format is ready to reconcile against bank statement transaction records.

Reconciliation Use Case

When a bank statement shows a check cleared for $1,247.50 on March 15, you can match it against the extracted check record to confirm payee, account routing, and the memo note — all without manually reviewing the scan. The reconciliation step happens in Excel or QuickBooks using the structured output from both the bank statement and check conversions.

MICR E-13B Font Recognition

MICR characters are printed in a specific machine-readable font. Standard OCR engines struggle with the unusual character shapes. Zera OCR includes a dedicated MICR recognition layer trained specifically on these characters, which is why routing and account number extraction accuracy is high even on lower-resolution scans.

Frequently Asked Questions

What does Zera Books extract from checks?

Zera Books extracts MICR line data (routing number, account number, check number), payee name, payment amount (numeric and written), date, memo line, and payer name from scanned checks.

Can Zera Books read MICR encoding from scanned checks?

Yes. Zera OCR is trained to recognize the MICR E-13B font used on the bottom of checks, extracting routing numbers, account numbers, and check numbers accurately from scanned images.

What format does Zera Books output check data in?

Check data is output to Excel or CSV with one row per check — routing number, account number, check number, payee, amount, date, and memo — ready for reconciliation against bank statements.
