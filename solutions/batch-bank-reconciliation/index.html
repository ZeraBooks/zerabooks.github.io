Running Batch Bank Reconciliation Across Multiple Clients and Accounts

The first week of each month is when reconciliation volume peaks. A practice with 40 clients each having 2–3 accounts can face 80–120 statements arriving in 48 hours. Processing them one at a time in a legacy tool isn't viable. Batch processing changes the math.

How Batch Processing Works in Zera Books

Zera Books accepts up to 50+ statement PDFs in a single upload. The processing queue handles them concurrently — you don't wait for one to finish before the next starts. As each file completes, the result appears in the dashboard. You can download completed exports while others are still processing.

95%+ auto-match rate: AI reconciliation matches transactions automatically. Unmatched items — unusual amounts, new payees, timing differences — are surfaced for manual review rather than buried in a flat list.

Duplicate Detection Across a Full Batch

When a client sends overlapping statements — say, a November statement and a combined Q4 statement — transactions appear in both files. Without duplicate detection, you'd import double the transactions into the accounting software without realizing it until the balance was wrong.

Zera Books' duplicate detection runs across all files in a batch and compares against the client's conversion history. It flags matches before export, not after import. You see exactly which transactions are duplicated and decide whether to exclude them.

Multi-Account Splitting in Batch

Clients don't always send separate files for each account. A combined PDF with checking, savings, and credit card data is common. Zera Books identifies account boundaries automatically and produces separate export files for each — even in a batch of 50+ uploads. You don't need to pre-sort documents before uploading.
