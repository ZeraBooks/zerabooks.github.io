Managing Client Documents and Conversions at Scale

When a bookkeeping practice grows past 15–20 clients, ad-hoc file organization stops working. Folders on a shared drive get messy. It becomes hard to know which version of a client's statement is current, or whether a particular document was already processed. Zera Books was designed with client management as a core feature, not an afterthought.

Client Folder Structure

Every client in Zera Books gets a dedicated folder in the dashboard. Within that folder, conversions are stored chronologically with original file name, upload date, and export format. You can pull up any past conversion without digging through email attachments or shared drives.

Client folders support tagging and notes, so you can flag documents that need review or record decisions about categorization choices for a specific client. This is useful when you have a staff bookkeeper doing the initial processing and a senior reviewing the output.

Conversion History and Version Control

Audit trail: Every conversion is timestamped and retained indefinitely. If a client disputes a figure, you can show exactly which source document produced which output, and when it was processed.

Version control matters when a client sends a corrected statement. You can upload the revision, compare the new output to the previous one, and see exactly which transactions changed. This catches issues before they propagate into the client's books.

Duplicate Detection Across the Client Folder

Clients often send overlapping statements — a December statement followed by a year-end combined PDF that includes December again. Zera Books' duplicate detection flags transactions that already appear in the folder, preventing double-counting without requiring manual cross-checking.

Workflow for Month-End Close

  1. Client sends PDFs via email or portal upload
  2. Upload all documents to the client's folder in bulk
  3. Zera Books processes and separates accounts automatically
  4. Review AI-categorized transactions, adjust any edge cases
  5. Export to the client's accounting software format
  6. Conversion stored in history for audit reference

The same workflow applies whether the client has one account or ten. Multi-account auto-detection handles the splitting — you don't manually separate documents before uploading.
