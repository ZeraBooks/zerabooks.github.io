CSV to OFX: Format Specification and Conversion Reference

OFX (Open Financial Exchange) is the open standard for financial data exchange, accepted by most accounting software that isn't QuickBooks. If you have transaction data in CSV and need to import it into Xero, Sage, MYOB, or Wave, OFX is usually the cleanest route.

OFX vs QBO: What's the Difference

QBO is Intuit's proprietary version of OFX. The core structure is identical — both use SGML-based tags for transactions — but QBO has Intuit-specific headers and is only accepted by QuickBooks products. OFX is the open standard accepted by everyone else. If you're importing into anything other than QuickBooks, use OFX.

OFX File Structure

An OFX 1.x file (the version most accounting software accepts) uses a header block followed by tagged transaction data:

OFXHEADER:100
DATA:OFXSGML
VERSION:151
SECURITY:NONE
ENCODING:USASCII
CHARSET:1252
COMPRESSION:NONE
OLDFILEUID:NONE
NEWFILEUID:NONE

<OFX>
  <BANKMSGSRSV1>
    <STMTTRNRS>
      <STMTRS>
        <BANKTRANLIST>
          <STMTTRN>
            <TRNTYPE>DEBIT
            <DTPOSTED>20250115120000
            <TRNAMT>-247.50
            <FITID>20250115-24750-0
            <NAME>Office Supplies Co
          </STMTTRN>
        </BANKTRANLIST>
      </STMTRS>
    </STMTTRNRS>
  </BANKMSGSRSV1>
</OFX>

Software compatibility: Xero, Sage 50, MYOB AccountRight, Wave, and Zoho Books all accept OFX 1.x. OFX 2.0 (XML-based) has narrower support — stick with OFX 1.x for the broadest compatibility.

Common Conversion Issues

  • Encoding errors: Special characters in descriptions (ampersands, accented letters) must be escaped or removed for valid SGML
  • Date format: DTPOSTED requires the time component — use YYYYMMDD120000, not just YYYYMMDD
  • Amount sign: Some CSV exports use absolute values with a separate debit/credit column — combine these before conversion

Frequently Asked Questions

What is OFX format?

OFX (Open Financial Exchange) is an open SGML-based standard for financial data. It's accepted by Xero, Sage, MYOB, Wave, and most non-Intuit accounting software.

What is the difference between OFX and QBO?

QBO is Intuit's proprietary OFX variant — same core structure, Intuit-specific headers. OFX is the open standard used by all other accounting platforms.

Which accounting software accepts OFX imports?

Xero, Sage, MYOB, Wave, and Zoho Books all accept OFX. QuickBooks accepts QBO (Intuit's variant) rather than standard OFX.
