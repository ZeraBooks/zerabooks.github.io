Converting Bank Statements to Xero-Compatible Import Format

Xero's bank reconciliation works best when transactions are imported directly rather than entered manually. For banks with direct feeds set up, this happens automatically. For banks without Xero connectivity — or for historical statements predating your feed — you need to convert the PDF to a format Xero accepts.

Formats Xero Accepts for Bank Import

Xero accepts three file types for manual bank statement import:

FormatNotes
CSVMost common; requires specific column mapping
OFXStandard financial exchange format; no column mapping needed
QIFOlder Quicken format; limited in Xero's newer interface

CSV is the most flexible since you can verify the data before import. OFX is faster for bulk imports because Xero parses the structure automatically.

Required CSV Columns for Xero

Xero's CSV import requires these columns at minimum:

  • Date — must match your Xero region's date format setting
  • Amount — positive for money in, negative for money out
  • Description — transaction narrative

Optional columns Xero also reads: Payee, Reference, Check Number. Zera Books outputs all of these when present in the source statement.

Date format tip: Xero's date format setting (DD/MM/YYYY vs MM/DD/YYYY) is set per organization. Zera Books' Xero export matches the format you select — make sure it aligns with your Xero organization's regional setting to avoid import errors.

Step-by-Step: Importing into Xero

  1. In Xero, go to Accounting → Bank Accounts
  2. Click the bank account you want to import into
  3. Click Import a statement
  4. Upload the CSV or OFX file from Zera Books
  5. For CSV: map the columns in Xero's import wizard
  6. Review the transaction count and date range, then click Import

Avoiding Duplicate Transactions in Xero

Xero uses date, amount, and description to detect potential duplicates during import. If you upload overlapping statements, Xero will flag likely duplicates for you to review before they're added to the feed. The AI-generated descriptions from Zera Books are consistent, which makes Xero's duplicate detection more reliable.

Frequently Asked Questions

What CSV format does Xero require for bank statement import?

At minimum: Date, Amount, Description. Optional: Payee, Reference, Check Number. Date format must match your Xero organization's regional setting.

Does Xero accept OFX files for bank import?

Yes. Xero accepts OFX and QIF in addition to CSV. OFX requires no column mapping since Xero parses the structure automatically.

How do I import a CSV bank statement into Xero?

Go to Accounting → Bank Accounts → Import a statement, upload the file, map columns (for CSV), and click Import.
