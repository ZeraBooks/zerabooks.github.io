IIF vs QBO: Which Format for Which QuickBooks?

QuickBooks comes in two fundamentally different products — Desktop and Online — and they use incompatible import formats. Using the wrong file type results in import errors or silent failures.

Criteria QBO Format IIF Format
Product QuickBooks Online QuickBooks Desktop (Pro/Premier/Enterprise)
Import Method Bank feeds (automatic match) File import: File → Utilities → Import → IIF Files
Account Mapping Match to existing transactions Pre-mapped to chart of accounts in the file
Categorization Rules-based after import Account field set at conversion time
Multi-line Transactions ✗ Single line only ✓ SPL rows for split entries
Memo Field Limited (MEMO tag) Full MEMO field per transaction and split

Key rule: If you use QuickBooks Desktop (installed software, not browser-based), you need IIF. If you use QuickBooks Online or QuickBooks Self-Employed, you need QBO or OFX format. Zera Books generates both — the right format is selected automatically based on your workflow.

IIF File Structure: What Zera Books Generates

IIF is a tab-delimited plain text format. Each transaction block contains a header row (!TRNS), one or more detail rows (!SPL), and a close row (!ENDTRNS). Here is what a valid IIF transaction block looks like after Zera Books processes your bank statement:

Sample IIF Output — Bank Statement Transaction
!TRNS	DATE	ACCNT	NAME	CLASS	AMOUNT	MEMO	CLEAR
!SPL	DATE	ACCNT	NAME	CLASS	AMOUNT	MEMO	CLEAR
!ENDTRNS

TRNS	02/14/2025	Business Checking	Stripe Payments		4250.00	Online payment deposit	N
SPL	02/14/2025	Revenue:Sales	Stripe Payments		-4250.00	Online payment deposit	N
ENDTRNS

TRNS	02/15/2025	Business Checking	Office Depot		-189.47	Office supplies purchase	N
SPL	02/15/2025	Expenses:Office Supplies	Office Depot		189.47	Office supplies purchase	N
ENDTRNS
Field Description Required? Example
DATE Transaction date in MM/DD/YYYY format (QB Desktop requirement) Required 02/14/2025
ACCNT Account name exactly as it appears in your QuickBooks Desktop chart of accounts Required Business Checking
NAME Vendor/payee name — used to match or create a QB vendor record Optional Office Depot
AMOUNT Signed amount. Positive = money in (TRNS row). Negative = offset (SPL row). Required -189.47
MEMO Transaction description extracted from bank statement. Preserved as-is. Optional Office supplies purchase
CLASS QuickBooks class for job/department tracking. Populated if classes are configured. Optional Operations
CLEAR Cleared status. N = not cleared. Y = cleared/reconciled. Optional N

The hardest part of IIF conversion is not the format — it is the ACCNT field. Every transaction in the TRNS row must reference an account name that exists exactly in your QuickBooks Desktop chart of accounts. Conversion Process: PDF Bank Statement to IIF

The conversion from bank statement PDF to a valid QuickBooks Desktop IIF file involves four stages. Here is exactly what happens at each step:

1

Upload + PDF Analysis

Upload your bank statement PDF (digital or scanned). For digital PDFs, Zera AI parses text directly. For scanned documents, Zera OCR processes the image first, achieving 95%+ accuracy on standard bank layouts including handwritten corrections and stamps.

2

Transaction Extraction + Normalization

Every transaction row is extracted: date, description, debit/credit amounts, and running balance. Dates are normalized to MM/DD/YYYY format required by QuickBooks Desktop. Descriptions are cleaned of encoding artifacts and truncated to IIF field limits.

3

Account Mapping + Categorization

Each transaction is assigned to an account in your QuickBooks Desktop chart. The AI reads vendor names, amounts, and memo text to determine the correct expense, revenue, or liability account. The TRNS row gets your bank account; the SPL row gets the offset account.

4

IIF Generation + Validation

The output IIF file is assembled with proper !TRNS/!SPL/!ENDTRNS structure, tab delimiters, and field ordering. A validation pass checks for account name accuracy, balanced amounts (TRNS + SPL = 0 per transaction), and date format compliance before download.

Frequently Asked Questions

What is an IIF file and why does QuickBooks Desktop use it?
+
IIF (Intuit Interchange Format) is a tab-delimited text format used by QuickBooks Desktop to import lists and transactions. It predates QBO format and is specifically designed for the desktop version. IIF files contain header rows (!TRNS, !SPL, !ENDTRNS) that define transaction types, accounts, amounts, and memo fields for direct import into QuickBooks Desktop without manual data entry.
Can I use QBO files instead of IIF for QuickBooks Desktop?
+
No — QBO format is for QuickBooks Online bank feeds, not QuickBooks Desktop. QuickBooks Desktop uses IIF for transaction imports. If you're running QuickBooks Desktop Pro, Premier, or Enterprise, you need IIF format for full transaction import with memo, split, and account information. QBO files will not import correctly into the desktop version.
How does Zera Books handle account mapping for IIF files?
+

Does Zera Books process scanned bank statement PDFs for IIF conversion?

Yes. Zera Books includes Zera OCR for scanned and image-based PDFs. The OCR layer achieves 95%+ accuracy on standard bank statement layouts.
