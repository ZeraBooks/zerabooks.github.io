Multi-Account Bank Statement Detection

Zera Books multi-account detection handles the common scenario where a client's bank sends a single combined PDF that includes multiple account types — checking, savings, credit card — all in one file. Instead of manually splitting the PDF before processing, Zera Books detects account boundaries automatically and outputs each account as a separate file.

No manual splitting required. Upload the combined PDF and receive separate files per account, each with the correct transactions.

How Detection Works

Zera AI reads account section headers, account numbers, and account type labels throughout the document. Banks typically indicate new account sections with headers like "Checking Account Summary" or show a new account number at the top of a new section. The AI identifies these boundaries and uses them to partition the transaction data.

The output file naming follows the account type and last-4 of the account number — for example, "Checking_1234.xlsx" and "Savings_5678.xlsx" — so there's no ambiguity about which file is which.

Why This Matters for Reconciliation

When all accounts land in one Excel file without separation, reconciliation against QuickBooks or Xero becomes more complex — you have to manually filter by account before importing. Separate output files per account map directly to separate accounts in your accounting software, which is how imports expect the data to arrive.

Multi-Account + Multi-Client

In a firm workflow, a client might have three combined PDFs from three different banks. Each PDF might contain two or three accounts. Zera Books processes all of them in one batch upload, splitting each into individual account files and organizing them under the client folder in the dashboard. No manual file management.

Yes. Zera Books automatically detects account boundaries within a combined PDF statement and outputs each account — checking, savings, credit card — as a separate Excel or CSV file.

How does Zera Books detect multiple accounts in one PDF?

Zera AI identifies account section headers, account numbers, and account type labels within the document. It uses these signals to determine where one account ends and another begins.
